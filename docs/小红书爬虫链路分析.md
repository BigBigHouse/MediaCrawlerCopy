## 二、获取原文（笔记详情）链路

### 2.1 入口：`core.py:search()` [行 132-189]

```
search()
  for keyword in config.KEYWORDS:
    while page <= max_pages:
      ① xhs_client.get_note_by_keyword()         # 搜索接口，返回笔记列表
      ② extract note_id, xsec_token               # 提取 ID 和安全令牌
      ③ asyncio.gather(get_note_detail_async_task) # 并发获取详情
      ④ xhs_store.update_xhs_note(note_detail)    # 存储笔记
      ⑤ batch_get_note_comments(note_ids)          # 获取评论
```

### 2.2 搜索接口：`client.py:get_note_by_keyword()` [行 271-301]

```
POST /api/sns/web/v1/search/notes

请求参数:
  keyword      搜索关键词
  page         页码（从 1 开始）
  page_size    每页条数（固定 20）
  search_id    搜索会话 ID（由 get_search_id() 生成）
  sort         排序方式（general / popularity_descending / time_descending）
  note_type    笔记类型（0=全部, 1=视频, 2=图文）

响应格式:
  {
    "items":    [...],   // 笔记列表
    "has_more": bool,    // 是否有更多
    "cursor":   "xxx"    // 分页游标
  }
```

### 2.3 获取详情：`core.py:get_note_detail_async_task()` [行 277-326]

```
主路径（API）:
  xhs_client.get_note_by_id()
    POST /api/sns/web/v1/feed
    参数: source_note_id, xsec_token, xsec_source, image_formats
    返回: note_detail（标题/正文/用户/互动数据/图片/视频）

降级路径（API 失败时，最多重试 3 次）:
  get_note_by_id_from_html()
    GET /explore/{note_id}
    通过 Playwright 渲染页面后从 HTML 解析
```

### 2.4 签名机制：`help.py:sign()` + `client.py:_pre_headers()` [行 74-113]

每次 API 请求前，通过 Playwright 的浏览器上下文注入 JS，调用小红书原生签名函数生成以下请求头：

- `X-S` — 主签名
- `X-T` — 时间戳
- `X-S-Common` — 公共签名参数

这是整个方案**不需要 JS 逆向**的核心：直接借用浏览器环境执行签名逻辑。

### 2.5 存储：`store/xhs/__init__.py:update_xhs_note()` [行 87-132]

原始 API 数据经过字段提取后，转换为扁平化存储格式：

```python
# 存储字段
{
  "note_id",          # 笔记 ID
  "type",             # image / video
  "title",            # 标题
  "desc",             # 正文
  "user_id",          # 作者 ID
  "nickname",         # 作者昵称
  "avatar",           # 头像 URL
  "liked_count",      # 点赞数
  "collected_count",  # 收藏数
  "comment_count",    # 评论数
  "share_count",      # 分享数
  "image_list",       # 图片 URL（逗号拼接）
  "tag_list",         # 话题标签（逗号拼接）
  "video_url",        # 视频地址
  "note_url",         # 完整笔记链接
  "source_keyword",   # 来源搜索词（从 source_keyword_var 读取）
  "xsec_token"
}
→ XhsStoreFactory.create_store().store_content()
→ 写入 DB / CSV / JSON / MongoDB 等
```

## 三、获取评论链路

### 3.1 入口：`core.py:batch_get_note_comments()` [行 328-343]

```python
for note_id in note_ids:
    asyncio.create_task(get_comments(note_id, xsec_token))
asyncio.gather(*task_list)   # 并发等待所有任务完成
```

### 3.2 首层评论分页：`client.py:get_note_all_comments()` [行 535-582]

```
cursor = ""
while has_more and len(all_comments) < max_count:

  GET /api/sns/web/v2/comment/page
  参数: note_id, cursor, xsec_token
  返回: { "comments": [...], "has_more": bool, "cursor": "xxx" }

  ① callback → xhs_store.batch_update_xhs_note_comments()  # 存储本页评论
  ② get_comments_all_sub_comments()                          # 获取二级评论
  ③ asyncio.sleep(crawl_interval)                            # 请求间隔
```

### 3.3 二级评论：`client.py:get_comments_all_sub_comments()` [行 584-665]

```
for comment in comments:
  if sub_comment_has_more:
    while has_more:
      GET /api/sns/web/v2/comment/sub/page
      参数: note_id, root_comment_id, cursor
      callback → store_comment()
```

控制开关：`config.ENABLE_GET_SUB_COMMENTS`，可独立关闭二级评论爬取。

### 3.4 存储：`store/xhs/__init__.py:update_xhs_note_comment()` [行 151-181]

```python
# 存储字段
{
  "comment_id",        # 评论 ID

  "content",           # 评论内容

  "nickname",          # 评论者昵称
  "avatar",            # 头像 URL
  "like_count",        # 评论点赞数
  "sub_comment_count", # 二级评论数
  "parent_comment_id", # 父评论 ID（二级评论关联用）

  "create_time",       # 发布时间戳
  "pictures"           # 评论图片 URL
```

## 四、完整数据流图

```
用户执行命令
    ↓
main.py → CrawlerFactory → XiaoHongShuCrawler.start()
    ↓
启动 Playwright 浏览器 → 登录（二维码 / Cookie / 手机号）
    ↓
search() / get_specified_notes() / get_creators_and_notes()
    ↓
[搜索] POST /search/notes → note_id 列表
    ↓
[并发，Semaphore 限制] get_note_detail_async_task()
    ├── POST /feed → note_detail
    │       ↓ 失败降级（重试 3 次后）
    │   GET /explore/{id} → HTML 解析
    ↓
update_xhs_note()   → store_content()  → DB / CSV / JSON / MongoDB
get_notice_media()  → 下载图片/视频   → data/xhs/images/ 或 data/xhs/videos/
    ↓
batch_get_note_comments()
    ↓
[并发] get_note_all_comments()
    ├── GET /comment/page     → 首层评论（游标分页）
    │       ↓ 每页回调
    │   batch_update_xhs_note_comments() → store_comment()
    └── GET /comment/sub/page → 二级评论（游标分页）
            ↓ 回调
        store_comment()
```

## 五、关键设计点

| 设计 | 位置 | 说明 |
|------|------|------|
| **Playwright 签名** | `client.py:_pre_headers()` | 不逆向 JS，在浏览器上下文直接调用原生签名函数 |
| **ContextVar** | `var.py:source_keyword_var` | 跨协程传递当前搜索词，写入存储用于数据溯源 |
| **Semaphore 并发控制** | `core.py:search()` | `MAX_CONCURRENCY_NUM`（默认 5）限制并发，避免触发封控 |
| **降级方案** | `core.py:get_note_detail_async_task()` | API 失败自动降级为 HTML 页面解析 |
| **两级评论** | `client.py` | 首层与二级评论用不同接口，独立游标分页 |
| **工厂模式存储** | `store/xhs/_store_impl.py` | `SAVE_DATA_OPTION` 运行时决定存储后端，支持 8 种 |
| **异步重试** | `client.py:request()` | 非致命错误最多重试 3 次，`NoteNotFoundError` 直接跳过 |

## 七、异常处理机制

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| `DataFetchError` | 接口返回非预期数据 | 记录错误日志，跳过该笔记 |
| `IPBlockError` | IP 被封控 | 触发代理轮换 |
| `NoteNotFoundError` | 笔记已删除或不可访问 | 记录警告日志，直接跳过 |
| `RetryError` | 重试 3 次仍失败 | 降级为 HTML 解析方案 |

## 八、关键配置参数

| 配置项 | 说明 |
|--------|------|
| `KEYWORDS` | 搜索关键词（逗号分隔） |
| `CRAWLER_MAX_NOTES_COUNT` | 最多爬取笔记数 |
| `CRAWLER_MAX_COMMENTS_COUNT_SINGLENOTES` | 单笔记最多评论数 |
| `MAX_CONCURRENCY_NUM` | 最大并发协程数（默认 5） |
| `CRAWLER_MAX_SLEEP_SEC` | 请求间隔秒数 |
| `SAVE_DATA_OPTION` | 存储方式（csv / json / db / mongodb 等） |
| `ENABLE_GET_COMMENTS` | 是否获取评论 |
| `ENABLE_GET_SUB_COMMENTS` | 是否获取二级评论 |
| `ENABLE_GET_MEIDAS` | 是否下载媒体文件 |
| `LOGIN_TYPE` | 登录方式（qrcode / phone / cookie） |
